Comparación de técnicas

1 Objetivo de aprendizaje

Aprender a leer un enunciado de Estructuras de Datos y Algoritmos para elegir una estructura o técnica adecuada. Al terminar esta página deberías poder:

  • identificar la operación dominante de un problema;
  • distinguir cuándo usar hash, heap, cola, lista, grafo o Union-Find;
  • separar problemas de búsqueda, optimización, restricciones y grafos;
  • comparar backtracking, greedy y programación dinámica sin confundir sus roles;
  • elegir algoritmos de grafos según la pregunta: recorrido, conectividad, ciclos, DAG, CFC, puentes, MST o caminos mínimos;
  • justificar complejidades por fases;
  • explicar por qué una alternativa típica no aplica.

La meta no es memorizar nombres. La meta es construir una respuesta defendible: qué estructura guarda qué información, qué operación acelera, qué algoritmo se aplica y por qué la complejidad queda razonablemente acotada.

2 Idea central del tema

En una interrogación, un mismo enunciado puede mezclar varias necesidades. Puede pedir acceso rápido por identificador, orden de llegada, conectividad, optimización o restricciones. La técnica correcta aparece cuando separas esas necesidades.

NotaIntuición

La elección de una estructura depende de la operación dominante. Si el problema consulta por llave, piensa en hash; si extrae por prioridad, piensa en heap; si procesa por orden de llegada, piensa en cola FIFO; si pregunta conectividad incremental, piensa en Union-Find.

La pregunta guía es:

¿Qué operación se repite muchas veces y qué estructura o algoritmo la resuelve naturalmente?

Por ejemplo, una tabla de hash sirve para acceso por llave, no para ordenar. Un heap sirve para obtener el mínimo o máximo por prioridad, no para buscar cualquier identificador de forma eficiente. Union-Find sirve para mantener componentes y responder conectividad incremental, no para reconstruir caminos.

3 Antes de partir: vocabulario mínimo

Concepto Significado para decidir técnica
Llave Identificador usado para acceder a un dato. Sugiere hash.
Prioridad Criterio para extraer mínimo o máximo. Sugiere heap.
FIFO Orden “primero en entrar, primero en salir”. Sugiere cola.
Restricción Regla que una solución no puede violar. Sugiere CSP/backtracking.
Dominio Valores posibles para una variable en un CSP.
Estado parcial Construcción incompleta que puede extenderse o descartarse.
Elección local Decisión tomada con información actual. Sugiere greedy si se puede justificar.
Subproblema Versión menor del problema original. Sugiere DP si se repite.
Recurrencia Relación que calcula un estado desde estados más pequeños.
Grafo Modelo de vértices y aristas para relaciones, rutas o dependencias.
DAG Grafo dirigido acíclico. Es requisito para orden topológico.
CFC Componente fuertemente conectada en un grafo dirigido.
MST Árbol de cobertura mínimo: conecta todos los vértices con costo total mínimo.
Camino mínimo Ruta de menor distancia o costo desde un origen. No es lo mismo que MST.
Componente Grupo de vértices conectados. Puede requerir BFS, DFS o Union-Find.

4 Cómo leer un enunciado

Antes de elegir una técnica, lee el problema en seis capas. Este orden evita saltar directo a un algoritmo por una palabra suelta.

Capa de lectura Pregunta que debes responder Señal típica
Objetivo ¿Qué pide producir o decidir? “encontrar”, “minimizar”, “verificar”, “clasificar”
Operaciones dominantes ¿Qué se hace muchas veces? buscar, insertar, extraer, unir, recorrer, asignar
Restricciones ¿Qué condiciones no se pueden romper? celdas prohibidas, no traslape, no ciclos, capacidad
Estructura de entrada ¿Los datos vienen como llaves, lista, aristas, matriz o eventos? (id, dato), aristas con costo, tareas y requisitos
Salida esperada ¿Pide valor, camino, conjunto, orden o todas las soluciones? distancia, padres, MST, asignación, tabla DP
Complejidad pedida ¿Hay límite de eficiencia o tamaño grande? n grande, muchas consultas, grafo disperso
TipSeñales del enunciado

Subraya verbos y sustantivos operacionales: consultar por identificador, atender al primero, extraer el menor, asignar bajo restricciones, conectar sedes, ordenar dependencias, calcular distancia o reconstruir camino.

Un buen diagnóstico no dice solo “uso grafo” o “uso DP”. Debe decir qué representan los vértices, aristas, estados, llaves, colas o componentes.

“todas las soluciones” Backtracking Debe explorar el espacio de soluciones.
“decisión local definitiva” Greedy Requiere demostrar elección segura.
“subproblemas repetidos”, p(i, S) Programación dinámica Se define estado, recurrencia y memoria.

“saber si dos elementos ya están conectados al agregar enlaces” Union-Find Mantiene componentes disjuntas.

6 Comparación por operación dominante

Cada estructura tiene una responsabilidad natural. Cuando un problema necesita más de una responsabilidad, se combinan estructuras.

Estructura Operación dominante Sirve para No sirve para
Hash Buscar, insertar o actualizar por llave Diccionarios, conteos, agrupación por identificador Mantener orden, sucesor, predecesor, mínimo global
Heap Extraer mínimo o máximo por prioridad Prioridades, top-k, eventos por urgencia Buscar cualquier llave en forma eficiente
Cola FIFO Atender por orden de llegada Filas, BFS, procesamiento por antigüedad Atender por prioridad
Lista Recorrer o enlazar elementos Secuencias, buckets de hash, inserción con referencia Acceso aleatorio eficiente por índice
Grafo Modelar relaciones Caminos, dependencias, ciclos, componentes, MST Acceso directo por llave sin apoyo adicional
Union-Find Unir componentes y consultar conectividad Kruskal, conectividad incremental Reconstruir caminos o calcular distancias
AdvertenciaMezcla de estructuras

Un enunciado puede requerir una estructura para acceso, otra para orden y otra para conectividad. Decir “uso hash + heap + grafo” solo es válido si explicas qué guarda cada una y qué operación resuelve.

Ejemplos frecuentes:

Necesidad mixta Combinación razonable Rol de cada parte
Atender por tipo y antigüedad Hash + cola FIFO Hash encuentra el tipo; cola conserva llegada.
Ranking con actualizaciones por cliente Hash + heap Hash acumula por cliente; heap entrega prioridad.
Grafo con nombres externos Hash + lista de adyacencia Hash traduce nombres; grafo guarda vecinos.
Kruskal Ordenar aristas + Union-Find Orden da greedy; Union-Find evita ciclos.

7 Backtracking, greedy y programación dinámica

Estas tres técnicas suelen aparecer en problemas de búsqueda u optimización, pero responden preguntas distintas.

Técnica Qué hace Cuándo usar Qué debe aparecer en la respuesta Riesgo típico
Backtracking Explora decisiones y retrocede Variables, dominios y restricciones CSP, caso base, validación, deshacer, podas Olvidar deshacer o no modelar restricciones
Greedy Toma decisiones locales definitivas Existe una elección local segura Candidatos, Select, Feasible, objetivo, prueba o contraejemplo Usarlo porque “suena natural” sin justificar
Programación dinámica Resuelve subproblemas y guarda resultados Hay recurrencia y subproblemas solapados Estado, caso base, recurrencia, orden, reconstrucción si aplica Dibujar una tabla sin definir qué significa

Backtracking parte modelando:

Parte del CSP Pregunta
Variables ¿Qué cosas debo asignar?
Dominios ¿Qué valores puede tomar cada variable?
Restricciones ¿Qué asignaciones son inválidas?
Objetivo ¿Busco existencia, todas las soluciones u optimización?

Greedy exige explicar por qué una decisión local puede quedar fija. Si no puedes justificar optimalidad, debes buscar un contraejemplo o cambiar de técnica.

Programación dinámica exige una definición precisa de estado. Una respuesta mínima debe incluir subproblema, recurrencia, caso base, orden de llenado y reconstrucción si el enunciado pide la solución concreta.

8 Grafos: elegir el algoritmo correcto

Modelar como grafo es solo el primer paso. Después debes distinguir qué pregunta se hace sobre el grafo.

Pregunta del enunciado Algoritmo recomendado Condición clave Resultado
Recorrer o marcar alcanzables BFS o DFS Grafo representado correctamente Visitados y, si se guarda, árbol de recorrido
Menor número de aristas desde un origen BFS Grafo no ponderado dist y parent
Profundizar, detectar ciclo dirigido DFS con colores Arista hacia gris indica ciclo Verdadero/falso o ciclo detectado
Ordenar tareas con dependencias TopSort Grafo dirigido acíclico Orden válido si es DAG
Componentes fuertemente conectadas Kosaraju Grafo dirigido Grupos CFC
Aristas cuya eliminación desconecta Puentes con DFS Grafo no dirigido Lista de puentes
Conectar todos con costo total mínimo Kruskal No dirigido ponderado MST o bosque mínimo
Conectar todos desde un vértice creciendo por costo Prim No dirigido ponderado MST
Distancia mínima con pesos no negativos Dijkstra Pesos no negativos Distancias desde un origen
ImportanteMST y caminos mínimos

MST no es lo mismo que caminos mínimos. Kruskal y Prim minimizan el costo total de conectar todos los vértices; BFS y Dijkstra minimizan distancia desde un origen hacia otros vértices.

Dos reglas que evitan confusiones:

  • BFS sirve para caminos mínimos no ponderados, porque todos los bordes cuentan lo mismo.
  • Dijkstra sirve para caminos mínimos con pesos no negativos, porque compara costos acumulados.

Union-Find aparece en Kruskal para evitar ciclos al agregar aristas. No guarda padres de caminos, no entrega rutas y no reemplaza BFS, DFS ni Dijkstra.

9 Pseudocódigo de diagnóstico

9.1 Diagnosticar una técnica

Input: enunciado de un problema.
Output: técnica candidata y razón principal.

DiagnosticarTecnica(enunciado):
    objetivo <- identificar que pide producir o decidir
    operaciones <- listar verbos que se repiten
    entrada <- identificar llaves, eventos, aristas, restricciones o estados

    if operaciones incluyen consulta por identificador:
        return "Hash", "acceso por llave"

    if operaciones incluyen extraer minimo o maximo repetidamente:
        return "Heap", "acceso por prioridad"

    if operaciones incluyen atender al primero que llego:
        return "Cola FIFO", "orden de llegada"

    if entrada contiene variables, dominios y restricciones:
        if objetivo pide todas las soluciones o existencia:
            return "Backtracking", "exploracion con validacion y deshacer"
        if aparecen estados repetidos:
            return "Programacion dinamica", "recurrencia con memoria"

    if objetivo pide optimizar con decisiones locales definitivas:
        if existe argumento de eleccion segura:
            return "Greedy", "decision local justificable"
        else:
            return "DP o backtracking", "greedy requiere prueba o contraejemplo"

    if entrada contiene vertices y relaciones:
        if objetivo pide distancia no ponderada:
            return "BFS", "niveles desde un origen"
        if objetivo pide ciclos, CFC, puentes u orden:
            return "DFS o variante", "estructura profunda del grafo"
        if objetivo pide conectar todos con costo minimo:
            return "Kruskal o Prim", "MST"
        if objetivo pide distancias ponderadas no negativas:
            return "Dijkstra", "caminos minimos"

    if operaciones incluyen unir componentes y consultar conectividad:
        return "Union-Find", "componentes disjuntas"

    return "Analizar combinacion", "puede requerir mas de una estructura"

9.2 Plantilla de respuesta de interrogación

Input: enunciado de desarrollo.
Output: respuesta ordenada con técnica, algoritmo, justificación y complejidad.

ResponderProblema(enunciado):
    escribir objetivo en una frase
    listar operaciones dominantes
    identificar estructura de entrada y salida pedida
    elegir estructura o tecnica principal

    si hay estructuras combinadas:
        explicar que guarda cada una
        explicar que operacion acelera cada una

    escribir algoritmo o pseudocodigo
    justificar por que responde el objetivo
    calcular complejidad por fases
    mencionar una alternativa incorrecta comun y por que falla

9.3 Elegir entre búsqueda y optimización

Input: problema de búsqueda, factibilidad u optimización.
Output: técnica más razonable entre backtracking, greedy y programación dinámica.

ElegirEntreBusquedaOptimizacion(problema):
    if hay variables, dominios y restricciones:
        if se piden todas las soluciones:
            return "Backtracking de enumeracion"
        if se pide existencia:
            return "Backtracking de decision con podas"
        if se pide optimizar una funcion objetivo:
            return "Backtracking de optimizacion o DP si hay estados repetidos"

    if hay decision local que no se reconsidera:
        if se puede demostrar eleccion segura:
            return "Greedy"
        else:
            return "Contraejemplo, DP o backtracking"

    if la solucion depende de subproblemas repetidos:
        definir estado
        definir caso base
        definir recurrencia
        definir orden de llenado
        return "Programacion dinamica"

    return "Modelar mejor el problema antes de elegir tecnica"

10 Complejidad: cómo justificar por fases

Una respuesta completa no solo escribe la cota final. Debe decir qué parte del algoritmo produce cada costo.

Usaremos n para cantidad de datos generales, K para tamaño máximo de dominio en backtracking, S para una suma o capacidad en DP, |V| para vértices y |E| para aristas.

Backtracking con n variables y dominio máximo K puede explorar:

\[ O(K^n) \]

Un recorrido de grafo con lista de adyacencia procesa vértices y aristas:

\[ O(|V| + |E|) \]

Kruskal suele estar dominado por ordenar aristas:

\[ O(|E| \log |E|) \]

Una DP de subconjuntos con n elementos y suma objetivo S suele tener:

\[ O(nS) \]

Fase Costo típico Cómo se justifica
Construir hash O(n) esperado Se procesa cada elemento una vez.
Buscar en hash O(1) esperado Depende de colisiones y factor de carga.
Insertar o extraer de heap O(log n) Se repara el árbol de heap.
Enqueue / Dequeue O(1) Operación en extremos de la cola.
Recorrer grafo con lista O(|V| + |E|) Cada vértice y arista se visita constante veces.
TopSort, Kosaraju, puentes O(|V| + |E|) Son variantes de DFS con estructuras auxiliares.
Kruskal O(|E| log |E|) Ordenar aristas domina.
Union-Find optimizado O(α(n)) amortizado Casi constante por operación.
Backtracking O(K^n) Tamaño del árbol de búsqueda.
DP n x S O(nS) Estados por costo de transición constante.
Greedy con ordenamiento O(n log n) Ordenar domina el barrido lineal.

11 Ejemplo guiado

11.1 Mini-enunciado 1: hash + cola

“Un sistema recibe solicitudes como (tipo, persona). Cuando se atiende un tipo, debe salir la persona que pidió ese tipo hace más tiempo.”

Aspecto Lectura
Señales del enunciado tipo actúa como llave; “hace más tiempo” exige FIFO dentro del tipo.
Técnica o estructura elegida Hash + cola FIFO.
Por qué aplica El hash encuentra la cola del tipo en O(1) esperado; la cola entrega al más antiguo.
Alternativa incorrecta Solo hash, porque permite llegar al tipo pero no conserva orden de llegada.

La estructura puede describirse como:

Hash[tipo] -> Cola FIFO de personas

11.2 Mini-enunciado 2: backtracking / CSP

“Hay que ubicar k estaciones en una grilla. Algunas celdas están prohibidas y todas las zonas prioritarias deben quedar cubiertas.”

Aspecto Lectura
Señales del enunciado Hay variables, dominios y restricciones.
Técnica o estructura elegida Backtracking / CSP.
Por qué aplica Cada estación es una variable; su dominio son celdas válidas; las restricciones descartan celdas prohibidas, repeticiones y coberturas incompletas.
Alternativa incorrecta Greedy sin prueba, porque elegir la celda que cubre más ahora puede impedir cubrir zonas futuras.

El modelamiento mínimo sería:

Parte Propuesta
Variables E_1, E_2, ..., E_k
Dominios Celdas disponibles de la grilla
Restricciones No prohibidas, no repetidas, cubrir zonas prioritarias
Objetivo Existencia, todas las soluciones u optimización según el enunciado

11.3 Mini-enunciado 3: Kruskal o Prim

“Se tienen sedes y conexiones posibles entre sedes, cada conexión con costo. Se quiere conectar todas las sedes minimizando el costo total.”

Aspecto Lectura
Señales del enunciado Conectar todos, costo por arista, minimizar costo total de la red.
Técnica o estructura elegida Kruskal o Prim.
Por qué aplica Es un MST sobre grafo no dirigido ponderado. Kruskal es directo si las aristas vienen como lista.
Alternativa incorrecta Dijkstra, porque Dijkstra calcula caminos mínimos desde un origen, no una red total mínima.

Kruskal además usa Union-Find para evitar ciclos:

if Find(u) != Find(v):
    agregar (u, v)
    Union(u, v)

11.4 Mini-enunciado 4: programación dinámica

“Dado un arreglo de números positivos, decidir si existe un subconjunto que suma exactamente S. En la recursión aparecen llamados repetidos del tipo p(i, S).”

Aspecto Lectura
Señales del enunciado Decisión tomar/no tomar; suma restante; estados repetidos p(i, S).
Técnica o estructura elegida Programación dinámica.
Por qué aplica Se define un estado por índice y suma restante, con recurrencia y memoización o tabla.
Alternativa incorrecta Backtracking puro, porque recalcula muchas veces los mismos estados.

Una recurrencia típica es:

\[ p(i, S) = p(i + 1, S - A[i]) \lor p(i + 1, S) \]

Aquí p(i, S) significa: “¿puedo formar suma S usando elementos desde la posición i en adelante?”.

12 Errores típicos

Error Por qué está mal Corrección
Usar hash para ordenar Hash no mantiene orden útil. Usar hash para llave y otra estructura para orden.
Usar heap para buscar cualquier id Heap solo garantiza acceso al mínimo o máximo. Usar hash para identificadores.
Usar cola cuando hay prioridad FIFO respeta llegada, no urgencia. Usar heap si el criterio es prioridad.
Usar Union-Find para reconstruir camino Union-Find solo sabe componentes. Usar BFS/DFS con parent para caminos.
Llamar greedy a cualquier algoritmo que ordena Ordenar no prueba optimalidad. Definir elección segura o entregar contraejemplo.
Usar backtracking sin variables ni dominios La recursión queda sin modelo. Escribir CSP: variables, dominios, restricciones.
Usar DP sin estado definido La tabla no tiene significado. Definir DP[...] en una frase precisa.
Aplicar TopSort en un grafo con ciclos El orden topológico solo existe en DAG. Detectar ciclos primero.
Usar Kruskal para caminos mínimos MST minimiza costo total de conexión, no rutas. Usar BFS o Dijkstra según pesos.
Usar BFS con pesos distintos BFS cuenta aristas, no costos distintos. Usar Dijkstra si los pesos son no negativos.

13 Problema representativo de interrogación

Ver enunciado

Para cada caso, clasifica la técnica o estructura más adecuada, justifica brevemente y menciona una alternativa incorrecta común.

  1. Un sistema debe consultar rápidamente si un estudiante tiene autorización usando su identificador único.
  2. Se deben asignar objetos a contenedores bajo restricciones de compatibilidad, capacidad y prohibiciones.
  3. Una universidad debe conectar sedes con enlaces de costo conocido minimizando el costo total de la red.
  4. En un grafo no ponderado, se pide la distancia mínima en número de aristas desde una sede inicial hasta todas las demás.
  5. Se quiere decidir si existe un subconjunto que suma S, y la formulación natural repite estados del tipo p(i, S).
  6. Se deben seleccionar actividades sin traslape. En una variante todas valen lo mismo; en otra cada actividad tiene una ganancia.
Ver solución
Caso Técnica adecuada Justificación Alternativa incorrecta común
Consulta por identificador único Hash La operación dominante es buscar por llave. Heap, porque no se consulta por prioridad.
Asignación de objetos bajo restricciones Backtracking / CSP Hay variables, dominios y restricciones; puede requerir existencia, todas las soluciones, podas o heurísticas. Greedy sin prueba, porque una asignación local puede bloquear una solución válida.
Conexión de sedes con costo mínimo Kruskal o Prim Se pide conectar todos los vértices minimizando costo total: MST. Dijkstra, porque no pide distancias desde un origen.
Distancia mínima en grafo no ponderado BFS Explora por niveles y entrega menor número de aristas. DFS, porque no garantiza distancia mínima.
Subconjuntos con recurrencia p(i, S) Programación dinámica Hay subproblemas repetidos por índice y suma restante. Backtracking puro, porque recalcula estados equivalentes.
Actividades con o sin ganancias Greedy si todas valen lo mismo; DP si tienen ganancias arbitrarias Sin ganancias, ordenar por término temprano maximiza cantidad; con ganancias, se compara tomar/no tomar. Greedy por mayor ganancia inmediata, porque puede bloquear una combinación mejor.

Una respuesta completa además debería decir qué se guarda. Por ejemplo, en el caso de autorizaciones:

Hash[id_estudiante] -> datos_de_autorizacion

En el caso de actividades con ganancias, una definición posible de estado es:

\[ OPT(j) = \text{máxima ganancia usando las primeras } j \text{ actividades} \]

La diferencia importante es que el problema sin ganancias maximiza cantidad; el problema con ganancias maximiza valor total.

14 Preguntas de repaso

  1. ¿Por qué la operación dominante es más importante que el nombre de la estructura?
  2. ¿Qué diferencia hay entre buscar por llave y buscar por prioridad?
  3. ¿Por qué hash no sirve para ordenar?
  4. ¿Por qué heap no permite buscar cualquier identificador eficientemente?
  5. ¿Qué garantiza una cola FIFO?
  6. ¿Qué responde Union-Find y qué no responde?
  7. ¿Qué tres elementos mínimos definen un CSP?
